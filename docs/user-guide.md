# 招投标自动化 Skill 用户手册

## 1. 产品介绍

### 1.1 是什么

招投标自动化 Skill 是一款基于 AI 的招投标全流程辅助工具，覆盖从招标信息监控、分析评估、投标文件生成、评标复盘到合同管理的完整业务流程。

### 1.2 解决什么问题

- **信息获取滞后**：人工搜索招标公告效率低，容易错过优质项目
- **投标准备周期长**：招标文件解析、投标文件撰写耗时耗力
- **文件质量不稳定**：人工编写的投标文件易出现遗漏、格式不统一
- **决策缺乏依据**：缺乏系统化的项目评估模型，靠经验判断风险高

### 1.3 目标用户

投标方（供应商/服务商），尤其适合：
- 需要持续跟踪多个招标平台的投标人
- 投标项目频次较高的企业
- 希望提升投标准备效率和质量的技术/商务团队

## 2. 核心能力（M1-M5）

### M1 Monitor - 招标信息监控

自动抓取多源招标公告，按照预设条件过滤符合要求的项目。

**支持数据源**：
- 中国政府采购网 (ccgp.gov.cn)
- 全国公共资源交易平台
- 各省市政府采购网
- 招标代理官网（省级以上）

**筛选维度**：
- 行业类别（IT/工程/咨询/货物）
- 预算金额区间
- 地区/省份
- 关键项目名称或关键词
- 招标方式（公开招标/邀请招标/竞争性磋商）

**输出**：招标信息摘要 → 进入评估流程

---

### M2 Analyze - 项目评估

对每条招标信息进行快速评估，判断是否值得投入资源投标。

**评估维度与权重**：

| 维度 | 权重 | 说明 |
|------|------|------|
| 竞争强度 | 20% | 历史中标数据、已报名供应商数量 |
| 资质匹配度 | 25% | 与我方资质证书的匹配程度 |
| 预算合理性 | 15% | 预算与市场行情对比 |
| 技术难度 | 20% | 需求复杂度 vs 我方技术储备 |
| 时间窗口 | 10% | 投标截止前剩余天数是否充足 |
| 利润空间 | 10% | 预算 vs 我方成本估算 |

**输出**：项目可行性评分 + 建议（投标/放弃/跟进）

---

### M3 Generate - 投标文件生成

根据招标文件要求，自动生成完整投标文件包。

**M3.1 技术投标书**
- 根据采购需求自动生成技术方案章节
- 引用我方相关项目案例
- 生成项目实施计划（甘特图格式）
- 编制质量保证措施

**M3.2 商务报价书**
- 根据预算、竞争态势、成本估算生成报价
- 提供单价分析表
- 输出三色报价区间（激进/标准/保守）

**M3.3 资格证明文件**
- 生成资质证书清单对照表
- 自动关联我方已有资质文件
- 标注缺失资质及办理建议

**输出**：`技术投标书.md` + `商务报价书.md` + `资格证明清单.md`

---

### M4 Evaluate - 评标辅助

投标后，根据评标方法论辅助复盘与改进。

**支持场景**：
- 知道中标结果后，分析评审打分明细
- 对照评标标准自评投标文件
- 识别失败原因，指导下次改进

**输出**：评标分析报告 + 改进建议清单

---

### M5 Contract - 合同管理

中标后，提取和管理合同关键条款。

**提取内容**：
- 合同金额与付款节点
- 履约保证金条款
- 违约责任条款
- 验收标准与节点
- 争议解决机制

**输出**：合同关键条款摘要 → 存入项目档案

## 3. 使用指南

### 3.1 如何启动监控

**第一步：配置监控参数**

编辑 `scripts/monitor_tenders.py` 中的筛选条件：

```python
FILTER_CONFIG = {
    "industry": ["IT", "工程"],
    "budget_range": [500000, 5000000],
    "region": ["北京", "上海", "广东"],
    "keywords": ["系统集成", "软件开发"],
    "bid_type": ["公开招标", "竞争性磋商"]
}
```

**第二步：运行监控脚本**

```bash
python scripts/monitor_tenders.py
```

**第三步：查看监控结果**

脚本运行后会输出符合条件的招标信息摘要，包含：
- 项目名称
- 采购单位
- 预算金额
- 截止时间
- 报名链接

> **建议**：监控脚本每日运行一次，招标截止前48小时内的项目会自动预警。

---

### 3.2 如何分析一个招标项目

**第一步：获取招标公告内容**

将招标公告文本或下载链接准备好。

**第二步：运行分析脚本**

```bash
python scripts/score_evaluation.py --tender <招标公告内容或文件路径>
```

**第三步：查看评估结果**

分析脚本会输出：
- 各维度评分（0-100分）
- 综合可行性评分
- 投标建议（投标/放弃/跟进）
- 关键风险点提示

**示例输出**：

```
项目可行性评分：78/100
建议：投标

维度评分：
- 竞争强度：65/100
- 资质匹配度：85/100
- 预算合理性：80/100
- 技术难度：70/100
- 时间窗口：90/100
- 利润空间：75/100

风险提示：
- 竞争对手较多，建议优化技术方案差异化
- 资质完全匹配，无需额外准备
```

---

### 3.3 如何生成投标文件

**第一步：准备招标文件**

确保已获取目标项目的招标文件（PDF或Word格式）。

**第二步：生成技术投标书**

```bash
python scripts/generate_technical.py --tender <招标文件路径> --output ./output/
```

**第三步：生成商务报价书**

```bash
python scripts/generate_price.py --tender <招标文件路径> --output ./output/
```

**第四步：生成资格证明清单**

```bash
python scripts/generate_technical.py --qualifications-only --output ./output/
```

**第五步：组装完整投标文件**

使用 `templates/` 下的模板清单检查完整性：
- 技术投标书 ✓
- 商务报价书 ✓
- 资格证明文件 ✓
- 授权委托书（如需）
- 投标保证金转账凭证（如需）

> **重要**：所有 AI 生成的文件必须经人工审核，确保符合最新政府采购法规要求。

---

### 3.4 如何进行合规性检查

**第一步：下载投标准备检查清单**

参考 `templates/bid-checklist.md` 中的完整检查项。

**第二步：对照招标文件自检**

1. **文件完整性**：逐项核对招标文件要求的每一项内容
2. **格式合规性**：检查签字盖章要求、文件格式、页数限制
3. **资质要求**：确认所有资质证书有效且在有效期内
4. **报价密封**：商务报价是否按要求密封、签字

**第三步：运行自动检查（可选）**

```bash
python scripts/generate_technical.py --check-only --tender <招标文件路径> --bid-files ./output/
```

## 4. 工作流程总览

```
[招标信息监控 M1]
       ↓
[快速评估 M2] → 决策：投标 / 放弃 / 需要人工判断
       ↓（投标决策）
[下载招标文件] → [解析招标要求]
       ↓
[技术方案生成 M3.1] + [商务报价生成 M3.2] + [资质文件准备 M3.3]
       ↓
[组装完整投标文件] → [投标准备检查]
       ↓
[提交投标]
       ↓
[评标辅助 M4]（获知结果后）
       ↓
[合同条款提取 M5]（中标后）
```

## 6. 注意事项

1. **政策合规**：所有生成的投标文件必须经人工审核，确保符合最新政府采购法规
2. **信息安全**：投标报价策略仅在本地运行，不上传外部服务
3. **时效性**：监控脚本建议每日运行，招标截止前48小时内的项目自动预警
4. **资质维护**：定期在 `references/` 下更新我方已有资质文件库
